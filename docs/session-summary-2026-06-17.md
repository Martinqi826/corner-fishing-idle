## 5. 社区鱼获照片墙 · 设计方案（`docs/community_photo_wall_design.html`）

- **创意（用户提出）**：玩家上传自己钓的鱼照片 → 后台审核通过 → 展示在该鱼图鉴里 + 点赞评论。一箭三雕：解决版权 + 社区感 + 强留存。
- **方案覆盖**：市场决策（Steam 全球主推）、架构（Godot ↔ Supabase ↔ 自动鉴黄/人工审核）、技术选型、数据模型、四条流程、法律/隐私/安全清单、成本、P0→P4 分阶段、决策清单。
- **三道闸（用户巧思 + 补强）**：
  - ① 上传门槛：Steam 付费即基线 + 进度解锁（认真玩家才有资格）；
  - ② 上传审核：AI 按**置信度分流**（自动过/自动拒/人工只看灰区）+ 信任用户免审；
  - ③ 举报审核：**加权阈值**（防组团刷举报）→ AI → 人工；**例外**：严重违法内容单条即立即处理。
  - 总发动机 = **信任分（声望）** 系统：自我调节地把人工审核压到最低。
- **建议**：当作「上线后第一个大版本」；详情卡先用「水彩 + 百科」上线，主体游戏先发 Steam。
