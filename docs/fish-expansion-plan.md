## 0. 一句话结论

- **不是"鱼不够"，是结构 + 存量没做实。** 现有 106 条里只有 28 条有正式图标、10 条有 lore。
- **钓鱼佬口径下，"值得做进游戏的鱼" ≈ 200–400 种**（IGFA 近 400、RF4 230+、Fishing Planet 200+、Fishbrain 300+）。我们从 106 扩到 **~200–250**，正好踩在硬核钓鱼游戏的市场验证甜点，且离 IGFA 上限有余量。
- **长留的真正杠杆是"水体多样性 + 破纪录文化"，不是"单塘鱼多"。** 重心：钓点 3 → **8–10 个**；新增**个人巨物纪录簿 + 钓鱼比赛事件**。
- **交付单位 = 主题钓点（每个带 25–40 条成套新鱼），不是散撒鱼名。**

## 1. 我们一共有多少种鱼可以用

### 1.1 生物学口径（噪音，仅作背景）

| 来源 | 规模 |
|---|---|
| FishBase | 35,000+ 种，含栖息地/营养级/体长体重字段 |
| Eschmeyer 分类目录 | 37,586 有效种（其中 19,250 淡水） |

结论：真实鱼种近乎无限，**不是约束**。当前 106 种只用了真实鱼库的 ~0.3%。

### 1.2 钓鱼佬口径（真正有用的数字）

把"全世界有多少鱼"换成"钓鱼圈真正在玩、在比、在记录的鱼有多少"，数字被天然筛到几百种：

| 口径 | 谁 | 数字 | 含义 |
|---|---|---|---|
| 全球运动钓最高权威 | IGFA 世界纪录 | **近 400 种**有 all-tackle 纪录 | 天花板锚点；乘 line-class/飞蝇/青年组 → 数千类目 |
| 硬核钓鱼游戏标杆 | 俄罗斯钓鱼4 (RF4) | **230+ 种** / ~18-21 个水体 | 公认鱼种最全 |
| 同上 | Fishing Planet | **200+ 种** | 季节/水温/底质驱动 |
| 钓鱼社区 App | Fishbrain（1500万钓友） | 识别 **300+ 热门运动钓鱼种** | 明确只收 sportfish |
| 细分圈子 | 美国黑鲈赛(B.A.S.S./MLF) | 核心 **3-4 种**黑鲈 | 整个职业巡回赛只围这几种 |
| 同上 | 中国路亚圈 | 核心 **~10-15 种** | 鲈/鳜/翘嘴/黑鱼/鳡/鲶/罗非/马口/红鲌/狗鱼/鲑 |
| 同上 | 欧洲鲤钓 / 海钓 / 矶钓 / 飞蝇 | 各几种~几十种 | 每个圈子有自己的小盘子 |

**两个关键观察：**

1. **黄金区间 = 200–400 种。** 这是被 IGFA（上限）和 RF4 / Fishing Planet（市场验证甜点）双重锚定的区间。我们的 ~200–250 目标稳。
2. **没有任何单一玩家/圈子长期接触超过几十种。** 钓鱼佬的"丰富感"来自**换水域 / 换圈子**，不是一个塘里鱼多——这直接决定了下面的设计重心。

## 2. 诊断：106 条为什么"感觉少"

```
品阶分布:  t0普通×21  t1优良×22  t2稀有×21  t3史诗×21  t4传说×14  t5神话×7
出鱼概率:    58%        25%        11%        4.5%       1.3%      0.2%
钓点:      河湾(river) 静水湖(lake) 海岸(coast)  —— 仅 3 个 active
```

1. **单池 churn 快**：每钓点抽 ~45 条，94% 落在 t0–t2 的 64 条里 → 一两小时常见层全见过。
2. **高阶看不见**：t4+t5 共 21 条只占 1.5%，多数玩家几乎遇不到，不计入丰富度体感。
3. **发现曲线前重后平**：常见行很快填满，之后只是"已知鱼换稀有度 roll"。
4. **收集深度隐形**：变体（×4）其实有 318 个收集格，但图鉴不显式摆出来 → 玩家感受不到"还有这么多没集"。
5. **缺"换水域"的新鲜感**：只有 3 个钓点，复刻不出钓鱼佬"今天去溪流、明天出海"的核心爽点。

> 核心：**留存 = 广度 × 深度 × 节奏（轮转）**。只加鱼名是最差的单一杠杆。

## 3. 设计原则

### 3.1 交付单位 = 主题钓点（不是散鱼）

**鱼永远是内容本体；钓点是把一批鱼"打包成新内容"的容器。每个新钓点自带 25–40 条成套新鱼。**

同样"做 30 条鱼"的两种交付方式对比：

| | 方式 A：散撒进现有钓点 | 方式 B：打包成新钓点（采纳） |
|---|---|---|
| 玩家体感 | 老池子 45→60，品阶概率不变 → 老常见鱼各自变罕见一点点（**稀释**） | 新背景/新招牌鱼/全空图鉴页/解锁目标 → **目的地** |
| 回访理由 | 无 | 有 |
| 美术 | 散落各品阶，难成套 | 一套连贯生态，好定调 |
| 像真钓鱼 | 否 | 是（换水域=换一套鱼） |

**"堆水体 > 堆鱼"的准确含义**：堆"换水域的新鲜感" > 堆"同一塘多几个鱼名"。鱼一条不少做，只是换装法。
（例外：往老钓点补鱼用于**填补瘦品阶层**，属"做厚存量"，是修补而非新内容。）

### 3.2 价值轴 vs 收集轴 分离

| 轴 | 成分 | 作用 |
|---|---|---|
| **价值轴**（驱动金币 = 经济跑步机） | 品阶 × 体重(线性) × 星级 × 钓点 value_mult × 鱼竿 | 短中期目标 |
| **收集/纪录轴**（驱动长留 = 金币封顶后的真正目标） | 鱼种 × 变体墙 × 词缀图鉴 × **巨物纪录** × 科属套装 × 限定鱼 | 长期目标 |

经济脊柱永远是 **品阶 × 体重**；怎么扩鱼都稳。

## 4. 规模与分期路线图

总量分期到 **~200–250 种**，以"钓点"为扩张单位，钓点 3 → **8–10 个**。

| 波次 | 新钓点（生态） | 新 tag | 候选真实鱼（举例） | 新增 | 累计 |
|---|---|---|---|---|---|
| 当前 | 河湾·静水湖·海岸 | river/lake/coast | （已有 106） | — | 106 |
| **P0** | （不加钓点，做厚存量） | — | 补 28→106 图标、变体墙显式可见、lore 扩到全 t2+ | 0 | 106 |

| **P2** | 河口红树林(汽水) · 热带珊瑚礁 | **brackish** · **reef** | 弹涂鱼/虱目鱼/射水鱼/四指马鲅／小丑鱼/刺尾鱼/鹦哥鱼/神仙鱼/苏眉(神话)/狮子鱼 | +60 | ~185 |

### 每钓点配额（让每个点都"全栈"）

约 12–15 条 t0/t1、8–10 条 t2、5–7 条 t3、2–3 条 t4、1–2 条 t5。哪里都能做梦钓神话，又靠招牌鱼区分性格。

## 5. 品阶结构

- **6 色品阶不动**（灰绿蓝紫橙红，见色识阶是行业最优解，**不加第 7 色**破坏认知）。
- **概率曲线 58/25/11/4.5/1.3/0.2 不动**；靠扩 roster 让每档成员够多，一个 session 内不重复见同一条 t3。
- **新增正交维度「限定鱼」**（不是第 7 色，沿用 6 色 + "限定"徽章）：只在特定**事件/天气/时段/季节**出现。例：满月夜才咬的鱼、春汛限定、节日锦鲤变体。把平铺图鉴变成"日历"，给"不同时间回来看看"的理由（动森月度鱼 / Fisch 活动鱼原理）。

## 6. 词缀系统：五层鱼牌模型

现有 3 条正交轴形式化为 5 层，并新增第 5 层"环境/特性词缀"。**纪律：避免乘法通胀。**

```
┌─ 价值轴 ────────────────────────────────────────────┐
│ ① 稀有度(品阶)   决定是哪条鱼、底价区间                  │
│ ② 体型(重量)     Fisch 线性插值，k²偏小个体  → 经济脊柱  │
│ ③ 品质(星级★)    鱼饵驱动，乘法 ≤×8                      │
│ ④ 变体(外观)     独立 roll，乘法 ≤×12，超稀有            │
└────────────────────────────────────────────────────┘
┌─ 收集/纪录轴 ──────────────────────────────────────┐
│ ④ 变体墙   斑斓/鎏金/七彩，106×3 = 318 收集格（头牌）    │
│ ⑤ 词缀(新) 环境/状态前缀，主要做"故事+图鉴+成就"        │
│ ·  巨物纪录 每种鱼最大重量进图鉴（钓鱼佬本命，见 §8）    │
│ ·  科属套装 按鲤科/鲑科/头足等成套，集齐给小 buff/外观   │
│ ·  限定收集 事件/季节鱼专属格                            │
└────────────────────────────────────────────────────┘
```

### 第 5 层「词缀/特性」具体设计

| 类别 | 例子 | 怎么 roll | 效果 |
|---|---|---|---|
| 环境系（钓获当下） | 「晨雾的」「寒潮中的」「月下的」「涨潮时的」「雷雨夜的」 | 由当前天气/时段/事件决定 | **加性** +5~15% + 一句氛围文案 |
| 状态系（鱼本身） | 「怀卵的」「带伤的」「年迈的(成精)」「迷途的(越界)」 | 体重/越界 habitat 触发 | 加性小幅 + 可选"放生得保育奖励" |
| 觅食系 | 「贪食的」「警觉的」 | 随机 | 影响咬钩/挣扎手感的 flavor |

**防通胀规则**：词缀走**加性、总和封顶 +40%**，主要服务图鉴/成就/故事；乘法大头永远只留给"星级×变体"两条超稀有轴。

> 理论封顶 jackpot ≈ 13000(神话底) × 1.4(词缀) × 8(完美) × 12(七彩) ≈ **175 万金币**，
> 0.0001% 的梦之一杆，做"个人成交纪录"高光即可，经济不会崩。

## 7. 体重与定价：保 Fisch 模型，新鱼卡带入价值带

新生态的鱼**必须落进现有同档价值带**，无论在哪钓经济都自洽：

```
t0: 1–18   t1: 14–55   t2: 55–190   t3: 200–580   t4: 750–2200   t5: 4500–13000
相邻 ×3~4，全谱 ×5000   ← 不动；新鱼按所属档塞进对应区间
```

- 钓点 `value_mult`（湖 1.06 / 海 1.12 / 未来深海·极地更高）负责"深水点更值钱"，不破坏带内数学。
- 巨物梦：`wmax` 按生态缩放——深海/极地/大洋点放真巨物（金枪/旗鱼/鲟鳇/大鲶），溪流点放小而精的冷水鱼。
- 经济锚点（现状，供新内容对齐）：鱼竿 cost = 200×2^(Lv-1)；背包 20→55 / 100→25000；
  鱼饵 0/800/5000/24000（星级）；鱼钩 0/2000/12000/60000（双钩）。

## 8. 三条留存线（对应钓鱼圈三种玩家）

| 玩家类型 | 留存线 | 内容 |
|---|---|---|
| 收集党 | 图鉴 / 变体墙 / 套装 | 318 格变体墙 + 科属套装 + 限定鱼 |
| **纪录党（钓鱼佬本命）** | **个人巨物纪录簿 + 钓鱼比赛事件** | 见下，**优先级提前到 P1** |
| 数值党 | 进阶 / 经济 | 鱼竿/饵/钩进阶 + 金币 |

### 8.1 个人巨物纪录簿（新）

- 每种鱼记录**个人最大重量**，进图鉴/详情卡（IGFA 世界纪录文化的家用化）。
- 破纪录时高光提示（"新纪录！"），是钓鱼佬最核心的 dopamine。

### 8.2 钓鱼比赛事件（新，可选）

- 限时事件：指定鱼种冲体重榜（如"本周末·翘嘴争霸"），到点结算给奖励。
- 单机版做"挑战自己历史最佳/AI 影子榜"；联网版可做真排行榜。
- 与现有 `event_data.gd` 事件框架同构，低干扰、契合"安静桌面陪伴"定位。

## 9. 美术节奏与质量门（"感觉多"的关键）

**原则：以"美术成套"为节拍，绝不放占位图进 runtime**（见 `docs/fish_icon_manifest.md` 质量门）。

- 每个 Phase 都跑 `tools/validate_game.gd`（逻辑必须全过）+ `tools/dev_screenshot.gd`（视觉人工把关）。
- 缺图标的鱼按品阶回退通用图标，但**不计入"已完成"**——宁可慢，不要一堆占位图。

## 10. 实施顺序（建议）

1. **P0 做厚存量**（零经济风险，最快见效）：补图标、变体墙显式可见、lore 扩写。
2. **P1**（最省成本，最对味）：上 `mountain_stream` + `deep_sea` 两钓点（tag/招牌鱼已就绪，各补 ~10 专属鱼）；**同步做个人巨物纪录簿框架**。
3. **P2**：河口红树林 + 珊瑚礁（珊瑚礁=颜值变体天堂，自带收集卖点）。
4. **P3**：极地/溶洞/野塘奇观生态 + 限定鱼框架 + 科属套装 + 钓鱼比赛事件。

存档：沿用现有版本化迁移（旧档默认落点 `river_bend`），新增字段向后兼容。
