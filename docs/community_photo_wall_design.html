Design Doc · 设计方案（评估用，未承诺开发）

社区鱼获照片墙 玩家上传自己钓的鱼照片 → 你后台审核 → 通过后展示在该鱼的图鉴里，可点赞评论。一个长在图鉴里的小社区。
本文档帮你在「建不建」之前看清：架构、技术、法律、成本、分几步走。

性质 从单机挂件 → 带后台的在线产品 主推路线 Steam 全球 + Supabase 建议 先 MVP 闭环，再加社交
Why · 为什么值得做

一箭三雕的好点子

🎯 解决版权

照片是玩家自己钓的、自愿上传，彻底绕开网络图的版权雷区——这正是之前"真实照片"卡住的根因。

🤝 社区感

点赞、评论、看别人的渔获，让一个安静的单机挂件长出"人味"，玩家之间有了连接。

🔁 强留存

为了上图鉴、为了被点赞，玩家会一次次回来——放置游戏最稀缺的长期回访动力。

🐟 内容自生长

106 种鱼的照片由社区慢慢填满，你不用一张张找图，内容随用户增长自动丰富。

Decision 0 · 最关键的前置判断

面向哪个市场？——它决定一切

UGC（用户上传内容）的技术栈和法律要求，因市场天差地别。这是必须先定的第一块多米诺骨牌。

🌍 Steam 全球 主推 · 契合你的上架计划

  • 后端用 Supabase（全球）即可，账号直接借 Steam，省掉自建登录与大半防滥用
  • 法律走 GDPR + Steam UGC 政策 + DMCA，有成熟范本
  • 已具备 Steam 国际收款条件 → 与此路线天然一致

🇨🇳 国内市场 要求重得多

  • UGC 平台需 ICP 备案 + 实名制 + 内容审核资质（网信办要求），个人主体很重
  • Supabase/Firebase 在国内访问不稳，需换 腾讯云/阿里云/微信云 等境内后端
  • 法律走 《网络安全法》《个人信息保护法》(PIPL)，UGC 审核是硬性义务
建议：以 Steam 全球 为主路线落地（与现有上架计划一致、合规负担最轻）；国内版作为以后单独评估的分支。下文按 Steam 全球路线展开。
Architecture · 架构总览

三块拼图：客户端 ↔ 后端 ↔ 审核

核心思想：游戏客户端只管"传 / 看 / 互动"；存储、数据库、账号、权限都交给 Supabase；审核先自动鉴黄打分，再由你人工终审。通过的才进 approved/、才会被游戏拉取展示。

Flows · 核心流程

四条主流程

📤 上传

图鉴某鱼详情卡点「上传我的渔获」
选本地照片 → 客户端压缩 + 抹 EXIF
传到 pending/，写一行 status=待审
自动鉴黄打分；明显违规直接拒

🛡️ 审核

你在审核台看待审队列（图+鉴黄分）
通过 → 转入 approved/、status=通过
驳回 → 标记原因，通知上传者

🖼️ 展示

详情卡拉该 fish_id 的 status=通过 照片墙
缩略图网格，点开看大图 + 上传者 + 赞/评

❤️ 点赞 / 💬 评论 / 🚩 举报

  • 点赞：一人一赞（唯一约束），实时累加；可做"本鱼最受欢迎"精选位
  • 评论：发表先入待审（或先发后审 + 关键词过滤），违规可删/封
  • 举报：任何人可举报不当内容 → 进你的复审队列 → 下架/封禁
Strategy · 门槛与审核策略

🔒 三道闸：把审核压力压到最低

你提的三个巧思非常对路——核心是「认真的用户才有资格，机器先扛掉大部分，人工只看真正模糊的」。下面把它们整合并各补强一点。

① 上传门槛：先证明你是认真玩家

  • Steam 付费即基线：游戏是买断制，每个能上传的人都已付费 + 有 Steam 实名身份——这本身就挡掉绝大多数机器人/水军（远胜免费 App）
  • 叠加进度门槛：再要求「钓获满 N 种 / 图鉴达成 ≥ X% / 游玩满 Y 小时」才解锁上传——既保证是真在玩的人，又让「上传资格」成为一种挣来的荣誉
  • （可选）创作者付费/更高等级：门槛高低你拍板，但别卡太死，否则照片墙长期空着（内容饥饿）
  • 限流：新用户每天少量上传，避免一次性灌水
补强提醒：门槛挡的是「垃圾量」，挡不住「内容真假」（付费玩家也可能传网图）——那交给下面的 AI＋人工。两者分工，别混为一谈。

② 上传审核：AI 先过 + 人工再过 —— 关键在「按置信度分流」

  • AI 第一道不只鉴黄，还判「这到底是不是鱼照片」（图像识别），顺手挡掉跑题图
  • 分流才是真省人工：明显安全且确实是鱼 → 可自动通过；明显违规 → 自动拒；只有中间灰区才进你的人工队列。AI 扛掉好判的 ~80%，你只看 ~20%
  • 信任用户免审：历史多次通过、零违规的老用户，后续可「先发后审」或免审——审核量随时间自然下降

③ 举报审核：达阈值才升级 —— 但有一类例外

  • 加权阈值，不是单纯数人头：按举报人信誉加权（老实人权重高、惯犯权重低），既挡单条恶意举报，也防组团刷举报恶意下架别人
  • 达阈值 → AI 复审 → 仍可疑才 人工终审，层层过滤
  • ⚠️ 例外（必须留）：涉黄涉暴涉违法这类严重内容不能等阈值——单条可信举报即立即临时隐藏 + 人工优先处理。纯阈值制对最坏内容有法律风险

🧩 串起来的发动机：信任分（声望）

给每个用户一个信任分：上传被通过 +分、被举报/驳回 −分。分高 → 免审、举报权重高、解锁更多额度；分低 → 严审、限流、最终封禁。这套自我调节的声望机制，才是长期把人工审核压到最低的真正发动机——你的三个巧思本质上都在喂养它。

Legal & Safety · 不能裸奔的部分

⚖️ 法律 / 隐私 / 安全清单

托管用户照片 = 你成了"平台"，有持续法律责任

这不是写完代码就完事，而是一份长期义务。下面每条都要落实，缺一个都可能是雷。

事项要做什么
用户协议 + 隐私政策明确：用户授权你展示其照片；禁止违法/侵权/色情/人身攻击内容；说明收集哪些数据、怎么用
EXIF 抹除上传前去掉照片里的 GPS 定位 / 设备信息（否则泄露用户家/钓点位置，隐私事故）
数据删除权用户能删自己的照片/账号（GDPR / PIPL 都要求"被遗忘权"）
内容审核义务先审后展示（已设计）；保留举报+下架通道；记录审核日志
侵权下架(DMCA)提供版权方投诉与快速下架流程（用户可能传了别人的图）
未成年人保护年龄声明；不收集儿童个人信息；评论防霸凌
防滥用上传/点赞/评论限流；一人一赞；封禁机制；垃圾/水军治理

建议：用户协议/隐私政策用成熟模板起草（Steam 上架本就需要隐私政策），上线前最好让懂行的人过一眼。

Roadmap · 分阶段

别一口气全做，分四步走

P0 · 决策与搭壳1 块
定市场/身份/区域；建 Supabase 项目、建表、配 RLS 权限、起草用户协议+隐私政策
P1 · MVP 最小闭环核心
上传 → 你审核 → 展示。先不做点赞评论。游戏端加联网+上传+照片墙；验证"真有人传"
P2 · 点赞 + 举报轻社交
一人一赞、最受欢迎精选位、用户举报通道。接自动鉴黄降人工
P3 · 评论 + 我的渔获社区感
评论（先审/先发后审）、个人主页"我上传的"、被赞通知
P4 · 社区运营长线
每周精选、排行榜、信任用户免审、活动；按数据决定加码
关键：P1 的 MVP 是分水岭——它能用最小代价回答"玩家到底会不会传图"。会，再往社交加码；不会，及时止损，不至于把点赞评论的复杂度白做。
In-Game · 游戏里怎么接

落在现成的详情卡上

  • 就在刚做的鱼种详情卡里加一块「🐟 社区渔获墙」：缩略图网格 + 「上传我的渔获」按钮
  • 详情卡的图片优先级可设：玩家精选照片 ＞ 自己曾上传 ＞ 水彩兜底（"百科↗"外链保留）
  • 需新增一个 联网模块（HTTPRequest 封装 + 账号态 + 缓存），这是游戏从"纯本地"迈向"联网"的第一步
  • 离线时优雅降级：拉不到照片墙就只显示水彩 + 百科，不影响挂机主体
Your Call · 你需要拍板的

决策清单

① 市场
Steam 全球（推荐）/ 国内 / 都要？ —— 决定技术栈与法律
② 身份
借 Steam 账号（推荐）/ 自建邮箱登录 / 匿名设备号（弱）？
③ 起步范围
先只做 P1 上传→审核→展示（推荐）还是一上来就带点赞评论？
④ 审核策略
全人工（你审）起步，何时上自动鉴黄 / 信任用户免审？
⑤ 法律
用户协议/隐私政策谁来起草、是否找人把关？
⑥ 商业化
照片墙纯免费社区，还是和付费/会员挂钩？（影响是否做、做多深）
⑦ 时机
现在就启动 P0，还是先把游戏主体上 Steam、回头再做这个大版本？
我的整体建议：把它当作游戏上线后的第一个大版本来规划，而不是发售前的拦路虎。先用「水彩+百科」让详情卡上线，主体游戏先发 Steam；社区照片墙按 P0→P1 起步，用 MVP 验证后再决定投入多深。
