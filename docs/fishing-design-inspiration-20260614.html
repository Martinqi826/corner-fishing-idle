Part One

第一部分 · 现实中的鱼竿垂钓

仅取「用鱼竿钓鱼」的范畴（路亚 / 台钓 / 海钓 / 飞蝇 / 冰钓 / 拖钓等），作为游戏机制的灵感来源。重点不是科普，而是「哪些真实细节能转译成挂机乐趣」。

1.1 要素拆解

真实垂钓是一套「组合系统」：玩家通过搭配装备 + 选择技法 + 适配环境，去博取一个本质上随机的结果。每一层都天然对应游戏里的一条成长线或一个变量——这正是钓鱼题材适合做成长 + 收集游戏的根本原因。

装备体系（每一件都可以是一条成长线）

部件作用游戏化映射
鱼竿 Rod决定抛投距离、灵敏度、硬度（调性），匹配目标鱼大小。本项目已有：缩短等待、提高高阶鱼权重、鱼价加成。核心金币去向。
渔轮 Reel收线速度与卸力（drag），搏大鱼时泄力防断线。可作为「搏鱼成功率 / 巨物上限」成长线，或并入鱼竿曲线避免系统过多。
钓线 Line线径 / 拉力值 / 隐蔽性，决定能搏多大鱼、惊不惊鱼。「断线风险 vs 渔获上限」的取舍变量，可做成可选难度而非硬墙。
鱼钩 Hook钩号大小、倒刺，影响中鱼率与鱼体大小。本项目已有：影响品相 / 通过率，第三条成长线。
浮漂 Float显示咬口信号（下沉 / 顿口 / 走漂）、控制钓棚深度。本项目核心反馈源：浮漂下沉 = 咬钩动效，是「偶尔操作」的天然提示点。
铅坠 Sinker控制饵到达的水层与抛投重量、稳定线组。可抽象为「钓层 / 水深」维度，区分不同鱼的栖息层。
饵 Bait活饵 / 商品饵：用气味与适口性吸引特定鱼。本项目已有：鱼饵 4 档，提高高品相通过率。可扩展为「针对性诱鱼」。
拟饵 Lure路亚假饵（亮片 / 软虫 / 米诺 / 铅笔等），靠动作模拟猎物。天然的「横向收集广度」素材：上百种拟饵 = 上百个可解锁的变体（参考 Fishing Clash 集卡）。

主要技法（玩法风味标签）

  • 抛投底钓 / 台钓：定点守钓，靠饵料窝点聚鱼——最接近「挂机等待」的范式。
  • 路亚 Lure fishing：不断抛收、用拟饵动作逗引掠食鱼——节奏快、主动、技巧天花板高。
  • 飞蝇 Fly fishing：抛投飞蝇线本身是门手艺，门槛与仪式感强。
  • 拖钓 Trolling：船拖假饵覆盖大水面——天然「移动 / 范围」概念。
  • 冰钓 / 矶钓 / 海钓：极端环境 = 天然的「钓点 / 赛季」差异化。

对象鱼与稀有度

  • 同一水域里，鱼天然分「常见小杂鱼 → 目标鱼 → 罕见巨物 / 记录鱼」的金字塔，本身就是稀有度系统。
  • 稀有不只看鱼种，还看个体：同种鱼的「破纪录重量」是终身追求（对应本项目「大 / 巨物」标记）。
  • 地域特有种、洄游季节限定种、保护鱼（需放流）——为「图鉴广度 + 赛季限定 + 伦理抉择」提供素材。

影响因素（"为什么今天钓不到"——不确定性的来源）

据钓鱼社区与海钓资料，水温、气压、潮汐、光照、浑浊度、水位变化、日月位置都会影响鱼的活性；且不同鱼种、不同水域反应不同，没有放之四海皆准的「最佳条件」。「气压/水温稳定数日 → 鱼吃口积极；冷锋过境后 24–48 小时 → 口慢」是较被认可的经验规律。

因素真实影响（经验向，个体差异大）游戏化潜力
季节洄游、繁殖、摄食窗口随季节变化。高 赛季化、限定鱼、内容轮换的天然骨架。
昼夜晨昏「黄金咬口窗口」普遍，部分鱼夜行。高 与真实时钟挂钩 = 低成本的"偶尔变化"。
潮汐海钓核心；涨退潮交替常是最佳摄食窗。高 本项目「海岸码头·涨潮」事件已用此。
天气 / 气压稳定优于剧变；冷锋后口慢。中 做成"事件 buff"即可，别做成劝退硬墙。
水温决定鱼代谢与活跃水层与时段。中 可与季节/钓点合并，避免变量爆炸。
水深 / 结构 / 水色鱼藏身于结构（沉木、坎、桥墩）；浑浊影响视饵。中 抽象为「钓点子区域 / 钓层」收集维度。
给本项目的提示：真实影响因素多到可以做一个硬核模拟器，但桌面挂件的克制原则是——把"因素"做成偶发的、正向的、看一眼就懂的事件 buff（晨雾 / 涨潮 / 鱼汛），而不是要求玩家天天查气压表的负担。本项目现有 7 事件方向完全正确。

1.2 钓鱼的完整循环

一次完整垂钓是「读水选点 → 配组合 → 抛投 → 等待/逗引 → 识别咬口 → 刺鱼 → 搏鱼 → 抄鱼 → 测量/拍照/留放」。对挂机游戏而言，关键在于识别哪几步该自动化、哪几步留给玩家做"偶尔的愉悦操作"。

读水选点

看水色、结构、鱼星，决定下竿位置。

配组合

按目标鱼配竿线钩饵漂坠。

抛投

把线组送到位，仪式感起点。

等待 / 逗引

守钓或路亚逗引，最磨人也最"挂机"。

识别咬口

看漂相、手感判断真假口。

刺鱼

抬竿中鱼，时机决定成败。

搏鱼

遛鱼控线防断防脱钩。

抄鱼

抄网上岸，最后一关也会翻车。

测量留放

称重拍照记录，留还是放。

映射到本项目（已基本实现）：「抛投→等待→咬钩→上鱼→结算」已覆盖前 5 步并自动化。真正值得设计的是第 6–9 步的"偶尔操作"潜力：①咬口时一次轻点"刺鱼"换品相加成（可选，不点也能自动收）；②结算时偶尔出现的「留 or 放」抉择（保护鱼 / 破纪录鱼）；③测量/图鉴纪录的成就感（已有"大/巨物/最大体重"）。详见第三部分 3.3。

1.3 为什么钓鱼让人上瘾、且能玩一辈子

这是对「超长线」目标最关键的一节。钓鱼之所以是终身爱好，不靠单一机制，而是多条独立的长期动机叠加，任何一条钝化了，还有其他几条拉着你回来。心理学层面，钓鱼被反复类比为"近似赌博"：抛投与收线像反复拉老虎机摇杆，每一次咬口都触发多巴胺，不确定奖励（variable reward）是核心成瘾源；同时身处自然、专注当下又带来减压与平静。

1. 不确定奖励

不知道下一杆是空军、小杂鱼还是破纪录巨物。这种间歇强化是钓鱼"近似赌博"的成瘾内核，也是挂机收益波动感的来源。

2. 技艺精进

读水、配组、抛投、控鱼……可终身打磨，永远有"更高的天花板"。给硬核玩家的纵向深度。

3. 收集

钓遍每个鱼种、每种拟饵、每个水域——「图鉴/收集癖」是收集型玩家的长期燃料（参见 Melvor 完成度日志、Stardew 83 鱼）。

4. 破纪录

同种鱼的最大个体是永无止境的目标（IGFA 世界纪录文化）。一个数字就能撑起几千小时追求。

5. 探索新水域

每片新水域 = 新鱼种 + 新打法 + 新风景。内容横向扩张的天然理由。

6. 装备追求

对竿、轮、饵的钻研与"剁手"本身就是乐趣，是付费意愿的现实映射。

7. 记录 / 日志

渔获日志、照片、数据复盘 → 自我量化的满足。对应游戏内的统计/图鉴/成就墙。

8. 放流与伦理

Catch & Release、保护鱼放生带来道德满足感，是一种"非数值"的正反馈。

9. 社交 / 比赛

钓友交流、钓鱼比赛、晒获——社群归属与竞争。对应排行榜/赛事（需谨慎，见下）。

对"超长线"的核心结论：不要指望靠"把数字调大"留住人。要像真实钓鱼一样，同时埋下多条相互独立的长期动机——收集广度（鱼种/拟饵/钓点）、破纪录（个体上限）、技艺天花板（掌握度）、装备追求、自我量化（日志/成就）。这样当一个玩家"鱼竿满级了"，他还有"图鉴没集齐""那条神话鱼还没破纪录""新钓点没解锁"在拉着他。

1.4 真实质感：值得做 vs 会让挂机变烦

真实钓鱼有很多"质感细节"，但桌面挂件的天职是低打扰。下面把真实细节分成"值得转译"和"会破坏挂机体验"两类。

值得做进游戏（增味、不增负担）

  • 空军是常态→ 收益波动、不保证每杆有鱼，但挂机版要"软化"：长期看一定有产出，避免真实那种几小时颗粒无收的挫败。
  • 晨昏黄金口 / 潮汐窗口→ 与真实时钟/事件挂钩的轻量正向 buff（已在做）。
  • 留与放的取舍→ 偶发的"放流换保育奖励 vs 卖钱"抉择，给一次有意义的轻点（本项目已有"保护鱼放流"事件）。
  • 破纪录的仪式感→ 巨物/新纪录时的特别反馈（特殊音效/图鉴金框），强化"难忘瞬间"。
  • 耐心被奖励→ 守得越久越可能出大物的"钓棚加深"感（参考 Tiny Fishing 越深越值钱）。

会让挂机变烦（应弱化或剔除）

  • 频繁精准操作：真实搏鱼要时刻控线，但挂件里若每条鱼都强制小游戏 = 灾难。最多做成"可选加成"。
  • 强制回访 / 错过即损失：真实"窗口期"若变成"不在线就亏"，等同体力惩罚，违背低打扰。
  • 装备损耗 / 频繁补给：真实要补线换钩，但游戏里做成消耗管理 = 微管理负担。
  • 过度拟真的失败：断线、脱钩、跑鱼在模拟器里是乐趣，在桌面挂件里是泼冷水，应极低频或可关。
  • 复杂配组门槛：真实配组很深，但不该成为新手必须查攻略才能玩的劝退点；做成"默认能玩、深挖有奖励"。

Part Two

第二部分 · 市面上的钓鱼游戏横评

按类型分别调研，每类提炼：核心要素 / 留存与长线机制 / 商业化模式 / 值得借鉴 / 坑。机制以公开资料为准，待核实 标注表示具体数值/实现需进一步确认，不在此编造。

2.1 休闲 / 挂机类

代表作：Tiny Fishing（网页休闲）、Cat Goes Fishing（Steam 休闲）、WEBFISHING（多人休闲社交）、各类 Fish Idle。与本项目最近的一类。

Tiny Fishing

核心要素：极简循环——抛线→收线接鱼→赚钱→升级（最大深度 / 一杆接鱼数 / 鱼价）→钓更深、更稀有的鱼。一个"5 分钟打开、20 分钟才舍得关"的小钩子。

长线机制："越深越值钱"的纵向阶梯 + 升级即时反馈，制造"再钓一杆"的冲动。

借鉴：升级与产出强绑定的即时正反馈；"深度"作为单一清晰的纵向进度轴。坑：深度玩穿后内容很快见底，缺横向广度与长期目标。

Cat Goes Fishing

核心要素：抛投 + 选饵 + 控线收鱼（收太猛会断线，是轻技巧点）；不同鱼有不同行为（深水/表层/掠食）。

长线机制：升级鱼竿解锁新区域与新鱼；里程碑明确——首条 Huge 鱼、最贵鱼、集齐图鉴（Catalog）。官方明言"没有真正的结局"。

借鉴：用"图鉴 + 里程碑 + 巨物"三件套撑长线，正好是本项目已有方向。坑：评测普遍指出久玩会重复，说明光靠"换更大鱼"不够，需要节奏变化。

WEBFISHING

核心要素：低门槛多人在线钓鱼，主打社交氛围与休闲社区（一起钓鱼、聊天、装扮）。

长线机制：社交本身 = 留存；收集与外观装扮。借鉴："轻社交 + 外观"是健康的非数值长线与变现点。坑：多人/服务器对一个单机桌面挂件是重负担，社交需谨慎裁剪（详见 3.1）。

Fish Idle / 放置钓鱼类

核心要素：典型放置框架——自动产出 + 升级倍率 + 离线收益 + 偶尔领取。

长线机制：多数靠 prestige/转生重置换永久乘数维持月活甚至年活；自动化是"留存保险"。商业化：常见激励视频 + 一次性去广告 + 加速礼包。坑：纯数字膨胀 + 强广告会快速透支好感。

2.2 拟真模拟类

代表作：Fishing Planet（F2P 模拟）、Russian Fishing 4（F2P 硬核模拟）、Call of the Wild: The Angler（买断开放世界）。它们与本项目气质相反（硬核、需专注），但其变现教训极有参考价值。

作品商业化关键观察 / 教训
Fishing Planet F2P + 内购 免费 + 付费内容（地图/装备/会员）。开放世界、深度拟真。借鉴：F2P 钓鱼 + 内容包是成立的商业模式。坑：硬核学习曲线对休闲玩家劝退，与桌面挂件定位冲突。
Russian Fishing 4 F2P + 高价内购 无主线、自由钓的硬核模拟，Steam 上意外热门，但变现广受差评：会员 3 天约 $3.5、1 个月约 $15，甚至传有约 $2000 的天价微交易（PC Gamer 报道）。教训（重要）：再好的核心玩法也会被"昂贵、像在卡你"的变现劝退。这是本项目要极力避免的反面教材。
Call of the Wild: The Angler 买断 + DLC 买断制开放世界钓鱼，氛围与画面取胜。借鉴：买断 + 可选 DLC 是被玩家接受度高的"公平"模型。坑：买断对一个想免费铺量的桌面挂件不直接适用，但其"DLC=内容包"思路可借。
一句话教训：拟真类证明了"钓鱼可以承载几千小时"，但 RF4 的差评也证明——长线再长，变现一旦显得"贵且在卡你"，口碑立刻崩。这条直接喂给第三部分的 F2P 边界。

2.3 玩法内嵌类

钓鱼不是主菜、而是大游戏里的一道：Stardew Valley、Animal Crossing、Dave the Diver、Sneaky Sasquatch。它们告诉我们：钓鱼的乐趣很大程度来自"钓之后"——收集、陈列、捐赠、订单，而非抛竿动作本身。

Stardew Valley

核心要素：有技巧的咬口小游戏（把鱼控在绿条内）；83 种鱼，集齐才能完成收藏并拿奖励。

长线机制：季节限定 + 天气限定 + 传奇鱼每档只能钓一次 → 收集需要跨季节、看天气、规划，天然拉长收集尾巴；传奇鱼可入鱼缸"陈列"而非卖钱。

借鉴："季节/天气限定 + 一次性传奇 + 收集墙"是低成本撑长线的黄金组合；钓上来后能"展示"比"卖掉"更有情感留存。

Animal Crossing

核心要素：极简钓鱼（找黑影→抛线→看时机收）；但"之后"很丰富：捐给博物馆、装饰小岛、季度钓鱼比赛。

长线机制：真实时钟/季节驱动鱼种轮换；博物馆陈列 = 收集成就的可视化；定期赛事制造回访理由（温和、非强制）。

借鉴："博物馆/陈列"把收集变成可炫耀的空间；与真实日历同步是几乎零成本的"内容感"。

Dave the Diver

核心要素：用鱼叉而非鱼竿（严格说不算鱼竿垂钓，仅作"采集→经营"循环参考）；白天潜水抓鱼，晚上经营寿司店。

长线机制："采集 → 经营/加工 → 再投资"的双循环让单一采集有了下游目的；叙事与多样玩法防腻。

借鉴：给"卖鱼"加一个下游（订单/加工/装点鱼缸），让渔获有"卖钱"之外的去处。坑：多系统耦合对极简挂件可能过重。

Sneaky Sasquatch

核心要素：开放世界休闲游戏里的钓鱼是众多活动之一，轻松、好上手。

长线机制：钓鱼作为"赚钱/调剂"的轻活动嵌在更大目标里。借鉴：钓鱼可以只是"舒适的背景活动"——这恰恰是桌面挂件的本质，不必把每个动作都做重。

2.4 治愈 / 收集 / 陈列类

与本项目气质最贴近的一类。代表：Chillquarium。核心不是"钓"，而是养 / 收集 / 陈列 / 看着舒服。

Chillquarium

核心要素：买随机鱼包 → 鱼随时间长大（1XP/秒，离线也长，右键喂食 +5XP 可加速）→ 养大卖钱 → 买更多包 / 更大鱼缸 / 装饰。纯治愈放置。

长线机制：收集（稀有鱼）+ 陈列（自定义鱼缸、装饰纯外观）+ 缓慢成长。QoL 做得到位：自动喂食、一键卖出已解锁鱼。

商业化：一次性买断（Steam 定价），无内购、无广告。

为什么它对本项目最有参考价值

  • 循环高度同构：本项目"钓→进鱼篓→卖→升级"≈ Chillquarium"买包→养大→卖→扩缸"，都是低压力的收集变现。
  • 离线成长是核心而非附属："不在线也在长"正是治愈挂机的灵魂——本项目离线收益方向一致。
  • QoL 即体验：自动喂食、一键卖出被当作"减负"而非付费墙——这正是本项目"自动卖鱼"该走的路（见 3.2）。
  • 装饰=纯外观长线：装饰不影响数值，却给收集型玩家无尽布置乐趣——健康的非 P2W 变现/留存点。
  • 坑：买断制意味着没有持续收入；本项目若要 F2P，需要在"不破坏治愈感"的前提下找到温和变现（见 3.2）。

2.5 手游 F2P 类

代表：Ace Fishing: Wild Catch、Fishing Clash、Fishing Strike。它们贡献了最成熟的F2P 长线 + 变现机制库，但也包含本项目应警惕的"激进套路"。

作品长线 / 留存机制变现与"坑"
Ace Fishing 60+ 钓点（海/河/湖/海岸/岛）环游世界；中鱼后是"拉力对抗"操作小游戏。体力/能量系统等具体机制待核实 F2P + 内购。坑：这类手游普遍用体力/能量限制单次游玩时长以促回访——与桌面挂件"持续陪伴"哲学直接冲突，不可照搬。
Fishing Clash 多重深度长线：拟饵集卡（CCG 式）并升级、每个钓点有技能树（用代币解锁加成）、每日 live 活动 + 每周随机事件、全球排行榜竞技。 F2P + 内购。借鉴：集卡=横向收集、技能树=纵向掌握、日/周活动=live-ops 节奏、排行榜=社交竞争——一整套长线模板。坑：集卡很容易滑向抽卡/付费强度，需守住"不卡进度"的底线。
Fishing Strike 500+ 鱼种收集、数十个世界区域、可升级/自定义渔船。 F2P + 内购。借鉴："超大收集池 + 区域扩张 + 载具/基地养成"是横向广度的极致样本。坑：体量越大越依赖抽卡/掉率变现，公平性风险随之上升。
提炼：手游 F2P 给了最丰富的"长线机制零件库"（集卡 / 技能树 / 日周活动 / 赛季 / 排行榜 / 超大图鉴），机制本身可借，但其"靠体力门 + 抽卡 + 卡进度逼氪"的驱动方式必须丢弃。本项目要做的是"拿走它的内容广度，换上 Chillquarium 的温和变现与治愈节奏"。

Part Three · 重点产出

第三部分 · 对《角落垂钓》的启示

把前两部分的素材，结构化映射到三个核心目的。每条都标注「是否适合一个安静的桌面挂机挂件」并给出取舍建议。

3.1 超长线内容设计（2000 / 5000 / 10000 小时不被消耗干净）

核心思路（来自 1.3 与各游戏长线机制）：不要靠单一轴硬撑，而要叠多条相互独立、消耗速率不同的长期动机。下表把可用机制按"是否适合安静桌面挂件"评级，并给取舍建议。

机制怎么用 / 参考来源适配度取舍建议
横向收集广度 更多鱼种 / 稀有变体（颜色/白化/季节型）/ 拟饵 / 钓点。参考 Fishing Strike 500+ 鱼、Fishing Clash 集卡。 非常适合 主力长线轴。当前 60 鱼可逐步扩，并引入"同种稀有变体"（白化/金色/季节限定）以极低美术成本翻倍图鉴深度。
稀有个体 / 破纪录 同种鱼的最大体重纪录（IGFA 文化）；巨物金框瞬间。本项目已有"大/巨物/最大体重"。 非常适合 把"个人最佳纪录"做成常驻可炫耀面板；偶发"超大个体"是免费的多巴胺事件。几乎零成本、无限尾巴。
掌握度 / 技巧天花板 Melvor 的逐项 Mastery、Fishing Clash 钓点技能树。每个鱼种/钓点有可累积的熟练度与小加成。 适合 做成"每钓点/每鱼种熟练度条 + 里程碑被动加成"，给硬核玩家纵向深度，且全程被动、不增操作负担。
纵向 Prestige / 转生 放置类标配：重置进度换永久乘数，制造"再爬一遍但更快"的月活/年活。 谨慎 可做"换竿/换钓场重开"式软转生，但治愈挂件慎用"清空式重置"——清掉图鉴/纪录会让收集党反感。建议只重置经济、保留收藏。
赛季 / Live-ops 与真实季节/节日挂钩的限定鱼、限定外观、限定事件。AC/Stardew 的季节限定、Fishing Clash 日周活动。 适合（轻量版） 性价比最高的"内容感"来源。与系统时钟同步即可自动轮换（冬季鱼、节日装饰），无需联网运营也能有"季节限定"。后续可加可选的在线活动。
沙盒 / 陈列布置 Chillquarium 鱼缸装饰、AC 博物馆、Stardew 鱼缸陈列。纯外观、无尽布置。 非常适合 强烈建议加入。让玩家把心爱渔获"陈列"在桌面角落场景里（标本墙/小鱼缸/装饰），契合"桌面摆件"本质，是健康的非数值长线 + 变现点。
程序化 / 涌现 随机事件、随机天气、随机个体属性制造"每天不一样"。本项目已有 7 事件 + 个体随机。 适合 继续扩事件库 + 个体变异维度；涌现感是低成本对抗"看穿"的关键。注意保持低干扰、正向为主。
世界纪录 / 排行榜 Fishing Clash 全球榜、IGFA 纪录文化。社交竞争的长线。 谨慎 可做异步、非压迫的榜（最大鱼周榜/好友榜），但避免实时 PvP 与排名焦虑——那会破坏"安静陪伴"。优先级低。
社交 / PvP WEBFISHING 社区、手游联赛。 基本不适合 不建议做实时多人/PvP。服务器成本高、与单机桌面挂件哲学冲突、引入竞争焦虑。最多保留"截图分享/晒图鉴"的轻社交。

推荐的"长线组合拳"（按优先级）

  1. 横向收集（鱼种 + 稀有变体 + 钓点）——主轴，消耗最慢。
  2. 陈列 / 布置沙盒——契合桌面摆件，给收集成果一个"家"。
  3. 破纪录 + 掌握度——零成本的无限尾巴 + 硬核纵深。
  4. 时钟驱动的轻赛季——免运营的"内容感"。
  5. 软转生（只重置经济、保留收藏）——给数值党可选的"再来一遍"。

明确不做 / 缓做（保护"安静"定位）

  • 实时多人、PvP、公会——重且违背哲学。
  • 清空式 Prestige（抹掉图鉴/纪录）——伤收集党。
  • 排名焦虑型实时排行榜——制造压迫。
  • 把"影响因素"做成需要查表的硬核模拟——增负担。
  • 体力/能量门——见 3.2，与挂机本质冲突。

3.2 公平 F2P 的边界（本体免费、便利付费）

行业数据：休闲/挂机玩家愿意为"身份认同、便利、个性化"付费——某统计显示放置类内购中超 58% 来自纯外观内容（无数值/无 P2W）；而激进付费墙、欺骗性机制、P2W 会直接招致差评、社媒声讨与大量卸载。同时，付费玩家本就稀少（休闲游戏付费率常 <5%），所以变现要靠"少数愿意支持的人买得开心"，而非"逼大众交钱"。

玩家"愿意"为什么付费（绿色区）

  • 外观 / 个性化：鱼竿皮肤、浮漂样式、场景主题（雨/夜/樱花角落）、鱼缸装饰、宠物。占放置类内购大头，零 P2W。
  • 一次性便利 / QoL：自动卖鱼、自动整理、一键收益——做成一次性买断最被接受（参考 Chillquarium 把自动喂食/一键卖出当免费 QoL；行业常见 ~$4.99 一次性去广告/解便利）。
  • 内容包 / DLC：新钓点世界包、新鱼系列、赛季外观集——"买内容"比"买进度"健康（参考 Angler 的 DLC、Fishing Planet 内容包）。
  • 支持者 / 打赏："支持开发者"礼包、署名感谢、专属外观——情感付费，口碑极好。
  • （可选）去广告：若有激励视频，提供一次性永久去广告。

玩家"反感"为什么付费（红色区）

  • 卡进度的硬墙："不付费就钓不动/产出腰斩"。
  • 体力 / 能量门：限制单次游玩、逼你买体力或等待——与"持续挂机陪伴"本质直接冲突，绝对不做。
  • 激进抽卡：靠低掉率 + 保底逼氪的拟饵/鱼抽卡。集卡机制可借，抽卡驱动不可取。
  • P2W：付费直接买数值碾压——破坏公平与收集成就感。
  • 把"本该免费的便利"做成反复付费墙：如背包扩容（见下）。

⚑ 针对委托人设想的"自动卖鱼 / 背包扩容收费"——风险评估与更好方案

① 自动卖鱼 → 较安全，但要选对形态

风险：低。自动化属于公认可付费的 QoL。但形态很关键。

方案：做成一次性买断的便利解锁（如"自动鱼贩"永久解锁），而非订阅/反复付费。可附"按品阶过滤的自动卖出规则"加深价值。手动卖鱼始终免费可用——付费只买"省事"，不买"能不能玩"。这与 Chillquarium 把"一键卖出"当免费 QoL 的思路一致；本项目把它做成可选付费 QoL 是温和且可接受的。

② 背包(鱼篓)扩容收费 → 高风险，像人为设卡

风险：高。容量上限本质上是开发者人为制造的瓶颈，再卖解药，玩家很容易识破为"先卡你、再收钱"，尤其它还会惩罚挂机（鱼篓满 = 离线/挂机收益被截断），与"低打扰陪伴"哲学正面冲突。

更好的替代设计：

  • 容量靠玩法免费成长（已有：升级鱼篓用金币）——保持纯免费路径，不设付费墙。
  • 满篓不再"硬截断"：满了自动把最低价鱼折价转化为金币（"自动鱼贩兜底"），让挂机永不空转——把痛点变成卖点。
  • 若一定要变现"容量"，包进外观+便利的"支持者礼包"里附赠，而不是单独当付费墙卖。

本项目 F2P 北极星原则（一句话）

"付费只买开心与省事（外观 / 一次性便利 / 内容包 / 支持开发者），永不买进度、永不设体力门、永不 P2W。" 让不付费的玩家觉得完整且被尊重，让愿意付费的少数人觉得"这是支持我喜欢的小东西"。这正是从 RF4 差评（反例）与 Chillquarium/外观经济（正例）两端得到的结论。

3.3 挂机 × 偶尔操作的节奏

目标：持续挂机为主，偶尔需要操作，但整体不让人觉得麻烦。判定一个"操作"该不该有的黄金标准——它是愉悦的（领取、决策、布置、欣赏）还是烦人的（频繁点击、强制回访、微管理、体力调度）。Rusty's Retirement 之所以成功（Steam 销量逾 50 万），正是因为它"嵌入你的生活，而不要求你嵌入它的世界"。

愉悦的"偶尔操作"（应保留/增加）

  • 领取式结算：回来一键收离线渔获/金币（多巴胺时刻，零压力）。
  • 有意义的抉择：偶发"留 or 放"（保护鱼/破纪录鱼）、"接哪个订单"——一次轻点、有取舍、不强制。
  • 布置 / 陈列：把心爱鱼摆进桌面鱼缸、换场景主题——慢节奏的自我表达。
  • 欣赏 / 翻图鉴：看新解锁、看个人纪录墙——纯正反馈。
  • 可选的小技巧点：咬口时一次轻点"刺鱼"换品相加成（不点也自动收，给主动玩家额外掌控感而非负担）。

烦人的操作（应避免）

  • 频繁点击：每条鱼都要手动收/手动卖——这正是"自动卖鱼"要解决的，应作为免费或一次性 QoL。
  • 强制回访 / 错过即损失：限时窗口"不在线就亏"= 变相体力惩罚。
  • 体力 / 能量管理：限制游玩时长——与挂机本质冲突，禁用。
  • 满篓硬截断：不在线鱼篓满了就停产 = 惩罚挂机，应自动兜底转化（见 3.2）。
  • 复杂配组/补给微管理：每次都要换线换钩配饵——做成"默认能跑，深挖有奖励"。

离线收益设计

本项目已有：最多 8 小时、效率 50%、受鱼篓余量限制。

  • 方向正确（治愈类离线成长是灵魂，Chillquarium 离线也长鱼）。
  • 建议：满篓不截断，自动折价兜底，让"挂一夜回来一定有惊喜"而非"满了就白挂"。
  • 回归时给一个温暖的"离线小结"卡片（钓了 N 条、最大一条是…），把领取做成仪式。
  • 8 小时上限是否放宽到 12–24h 可做 A/B：上限太短会隐性逼日活（玩家社区对"逼每日登录"有怨言）。

自动化分层

把自动化做成"可成长的层级"，既是长线也是变现点：

  • L0 手动：手动抛/收/卖（始终免费）。
  • L1 自动钓：已实现的自动循环。
  • L2 自动卖（按规则）：一次性便利解锁，可设过滤。
  • L3 自动整理/兜底：满篓折价、自动锁稀有。
  • 关键：自动化是"减负"，永不是"不付费就玩不动"。

"结算 / 抉择"节点设计

需要一次轻点的节点，要稀疏、正向、可跳过：

  • 频率克制：抉择类事件低频出现，避免变成"不停弹窗"。
  • 都给默认安全项：不点也有合理结果（自动收/默认卖），点了有额外收益或情感价值。
  • 低干扰呈现：沿用现有 HUD 角标/柔和提示，不抢屏、不强制暂停玩家工作。
  • 奖励"回来看一眼"而非"必须盯着"。
节奏总原则：像 Rusty's Retirement 那样——默认完全可以"开着不管"，但每次"回来看一眼"都有愉悦的小事可做（领取、一个抉择、摆个装饰、翻翻新纪录）。操作是"邀请"而非"义务"，是奖励主动玩家、而不是惩罚被动玩家。
